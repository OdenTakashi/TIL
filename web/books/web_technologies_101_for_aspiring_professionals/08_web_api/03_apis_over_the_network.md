## ネットワーク越しのAPI
RPC(Remote procedure call/遠隔手続き呼び出し): 分散システム上のコンピュータが互いの機能を呼び出すこと
  - CORBA(Common Object Request Broker Architecture/コルバ)
    - 広く普及したRPC技術
    - 1990 ~ 2000
      - インターネット本格普及前
    - 仮数呼び出しのような使い勝手になることが期待されていた
    - ネットワーク越しに行うため特有の複雑性が生じる
      - 通信の遅延
      - 相手コンピュータのダウン
    - IDL
      - 遠隔呼び出しの処理しよう記述ファイル
      - 基にスタブ、スケルトンと呼ばれる通信処理を担うプログラムを生成
        - rescue 処理をスタブに寄せれる
        - 利用する側からローカルAPIと同等に見える

## SOAP
- Web技術ベースのRPC実現手段
- 通信プロトコルを限定せずに(HTTP, SMTP..)使えることを目指したが、HTTPで利用されることがほとんどだった
  - HTTPメインで使われたため、ファイアーウォールを超えやすかった
- XMPで表現される
  - 簡単な通信でも情報量が多かった
